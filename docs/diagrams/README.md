# AgentHive Draw.io Diagrams

This folder contains separate architecture diagrams.

## Files

1. `backend-flow.drawio` - backend request and agent orchestration flow.
2. `frontend-flow.drawio` - frontend dashboard structure and UI data flow.
3. `frontend-backend-connection.drawio` - connection between dashboard, API, workers, DB, Redis, MinIO, and LLMs.
4. `database-design.drawio` - database table grouping and relationships.
5. `redis-cache-queue-flow.drawio` - Redis as cache, queue, idempotency, and dead-letter flow.
6. `llm-router-fallback.drawio` - Gemini primary, Ollama fallback, and other provider fallback flow.
7. `request-data-flow.drawio` - user request to final response and logs.
8. `developer-deployment-flow.drawio` - developer-focused Docker/dev/prod flow.
9. `full-system-architecture.drawio` - complete all-in-one architecture.

Open these files in diagrams.net / draw.io.
